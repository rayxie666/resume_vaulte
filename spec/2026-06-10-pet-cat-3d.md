# Spec: 3D 宠物猫「工坊猫」—— 写简历时的桌面陪伴

- 日期:2026-06-10
- 前置:`spec/2026-06-10-ui-redesign-letterpress.md`(品牌:深夜排印工坊)、
  `spec/2026-06-10-motion-polish.md`(动效 token 与纪律)
- 目标:在界面右下角常驻一只**真实风格**的 3D 猫(能看清身上的猫毛),
  陪伴用户写简历。它跟随鼠标视线、对键盘敲击有反应、可以被抚摸,会主动
  发出克制的互动邀请;在保存 / 打 checkpoint / 编译成功时为用户庆祝。
- 状态:**已实装(2026-06-10)。** 模型采用程序化构建(用户拍板,
  替代 §11 的 GLB 采购方案):参数化身体 + shell fur 壳层毛发 + 程序化
  动画,外观对照用户提供的长毛虎斑幼猫参考照(金面、灰棕虎斑、奶白
  胸毛、大圆绿眼)。调试工具:`pet-dev.html` + `src/pet/devMain.ts`
  (仅 dev server 可访问,不进产物)。

---

## 1. 概念与定位

每一间深夜亮灯的排印工坊都该有一只猫。它睡在灯桌边上,你敲字它看着,
你保存它高兴。这与「安静直到出事」的产品原则不冲突——猫本身就是
"出事"之外唯一被允许的生命体,但它必须遵守工坊的规矩:

1. **猫是陪伴,不是吉祥物**:不挡内容、不抢焦点、永远可以一键关掉。
2. **真实风格**:写实比例与毛发(shell fur,肉眼可见的毛层),不做
   卡通大头猫;毛色用工坊色系——暖棕虎斑(brass 同族色相),融入主题。
3. **行为优先级**:用户操作的反馈 > 庆祝 > 自发行为(邀请/理毛/睡觉)。
   任何自发行为可被用户操作立刻打断。

## 2. 技术选型

| 项 | 决定 | 理由 |
|---|---|---|
| 渲染 | three.js(`three` + `GLTFLoader`,动态 `import()` 懒加载) | WKWebView 支持 WebGL2;按需加载不增首屏;社区毛发方案成熟 |
| 毛发 | **Shell fur**(壳层毛发):基础猫模型沿法线挤出 12–16 层实例化壳,alpha 噪声纹理逐层收缩 | 真实感毛发在实时端的标准做法;一个 InstancedMesh 一次 draw call;远胜贴图猫 |
| 模型 | 带骨骼的写实猫 GLB(要求见 §11),`public/pet/cat.glb` | 自建模型工程量不可控,采购/CC 资产 + 命名规范的动画 clip |
| 动画 | `AnimationMixer` clip 交叉淡入(180ms)+ 程序化头/眼骨骼 look-at、程序化眨眼 | 视线跟随必须程序化,clip 只管身体 |
| 接入 | 独立 `<PetOverlay>` 组件 + 轻量事件总线 `petEvents` | App.tsx 各保存点只加一行 `emit()`,猫与业务零耦合 |

WKWebView 注意:pdfjs 的崩溃(91825e6)源于缺失的运行时 API
(`Map.prototype.getOrInsertComputed`),非语法问题;three.js 不依赖
此类新 API,Vite 默认 target 即可。实装时仍须在真机 WKWebView 验证
WebGL2 上下文创建失败的兜底(见 §10-6)。

## 3. 呈现与布局

- **容器**:`position: fixed`,默认右下角(`bottom: 16px; right: 16px`),
  画布 200×240(逻辑 px),`--z-pet: 55`(modal 50 之上、toast 60 之下
  ——modal 打开时猫仍可见但变安静,见 §4)。
- **命中区**:canvas 本身 `pointer-events: none`(不挡任何 UI 点击);
  其上叠一个跟随猫身体包围盒的透明命中 div(约 120×100,随状态微调),
  只有它接收 hover / click / 拖拽。**猫以外的画布区域全部点击穿透。**
- **可拖拽换位**:按住猫拖动 ≥ 8px 进入搬运模式(猫播放被拎起的 clip),
  松手吸附到最近的四个角,位置持久化 `localStorage["rv-pet-corner"]`。
  位移仅 transform,带 `--t-med` 吸附动画。
- **遮挡让位**:猫所在角落出现 `select-bar` / `sync-badge` / 编译错误
  抽屉时,猫容器临时上移让出空间(transform,`--t-med`),消失后归位。
- 窗口 resize:fixed + 角落锚定天然适应;min 720×480 下不缩放、不重排。

## 4. 状态机

```
sleep ←→ idle ←→ watch(视线跟随) → petted(被抚摸)
              ↘ typing(看打字)   → invite(互动邀请)
              ↘ celebrate(庆祝)  → concerned(编译失败)
```

| 状态 | 进入条件 | 表现 |
|---|---|---|
| `idle` | 默认 | 坐姿呼吸 loop,偶发耳动、尾尖摆动、程序化眨眼(2–6s 随机) |
| `watch` | 鼠标在窗口内移动 | 头 + 眼骨骼 look-at 鼠标投影点(lerp 平滑,头部偏转限 ±35°);鼠标快速晃动时瞳孔放大 |
| `typing` | `typing` 事件(节流 1s)持续触发 | 转头看向编辑器一侧,尾尖随击键节奏轻拍;停止 4s 后回 `idle` |
| `petted` | 指针在命中区内往复滑动 ≥ 2 次 | 眯眼蹭手 clip + 身体随指针方向微倾;持续抚摸进入呼噜(身体 2Hz 微颤);离开 2s 后回 `idle` |
| `sleep` | 无任何输入(全局 mousemove/keydown)≥ 5 分钟 | 卧姿睡觉 loop,呼吸起伏;任意输入 → 伸懒腰 clip 唤醒 → `idle` |
| `invite` | 见 §6 | 单次行为,完成即回 `idle` |
| `celebrate` | 见 §7 | 单次行为,完成即回 `idle` |
| `concerned` | `compile-error` 事件 | 歪头看向预览一侧、耳朵后压一次,然后安静坐着(不循环打扰);`compiled` 事件解除 |
| 静默 | 任意 modal 打开期间 | 不发起 invite / celebrate(排队丢弃),仅保留呼吸与视线 |

所有 clip 切换 180ms 交叉淡入;被打断的自发行为直接淡出,不强制播完。

## 5. 互动(鼠标 / 键盘)

- **视线跟随**:监听 `document` 级 `mousemove`(被动监听,不占命中),
  鼠标屏幕坐标 → 猫头部前方虚拟平面投影点 → 头/眼骨骼四元数 slerp
  (头 0.08、眼 0.2 的 lerp 系数,眼比头快——猫的真实行为)。
- **抚摸**:命中区内按下并滑动 = 摸;短按一下 = 戳(猫甩一下耳朵)。
  呼噜声**默认关闭**,设置里可开(音量极低的 purr loop)。
- **键盘**:不监听全局键盘内容,只接收 `CodeEditor onChange` 节流后的
  `typing` 事件(无任何按键内容传递,纯活动信号)。
- **拖拽**:见 §3。

## 6. 互动邀请(克制规则)

猫可以主动邀请,但频率受三重闸门约束:

1. **冷却**:两次自发行为(邀请 + 理毛等小动作)间隔 ≥ 10 分钟(随机
   10–18 分钟),App 启动后首 5 分钟不邀请。
2. **时机**:仅在用户**正在活跃**(近 30s 有输入)且**无 modal、无编译
   错误**时发起——猫找人玩,不吵睡着的工坊。
3. **形式**(随机其一,全程 ≤ 4s,不发系统通知、不出声):
   - 朝用户方向抬爪虚拍两下 + 头顶冒一个 `喵?` 气泡(brass 描边、
     serif 字体的小气泡,跟随主题色,2.5s 自动消失);
   - 站起来伸个懒腰再坐下(顺带提醒用户也该伸展了——连续 `typing`
     超过 45 分钟时此形式概率加权);
   - 追自己尾巴半圈。
4. 邀请后 6s 内用户 hover/抚摸 = 接受,猫进入 `petted` 并重置冷却;
   无响应 = 安静回 `idle`,下次冷却时间 ×1.5(猫也会识趣)。

## 7. 保存庆祝(分级,带疲劳保护)

| 触发事件 | 来源 | 庆祝规格 |
|---|---|---|
| `saved`(手动保存) | `handleSave()` 成功后 | **小**:坐直 + 尾巴竖起摇两下 + 满足地眯眼,~1.5s |
| `compiled`(PDF 编译成功) | `LatexPreview.onRendered` | **小**:转头看预览,跟着「落纸」hero 时刻盯一眼新页,尾尖一翘;与落纸 240ms 联动但不抢戏 |
| `checkpoint`(打快照成功) | `handleCheckpoint()` 成功后 | **大**:起身原地小跳 + 画布内一撮 brass 色光尘从猫身边扬起(粒子仅存在于猫的 200×240 画布内,不进 UI 层,不违反动效 spec 的"禁彩带"——那条红线管的是产品 UI),~2.5s |
| `checkpoint` + GitHub push 成功 | 同上链路的 push 回调 | 大庆祝结尾追加:抬头望向 sync-badge 方向点一下头 |

疲劳保护:同级庆祝 60s 冷却,冷却内重复触发降级为眨眼 + 尾尖一动;
连续保存刷庆祝刷不出来——仪式感靠稀缺。

## 8. 事件总线与接入点

新建 `src/pet/petEvents.ts`(~30 行 pub/sub,无依赖):

```ts
type PetEvent = "saved" | "checkpoint" | "pushed" | "compiled"
              | "compile-error" | "restored" | "typing";
petEvents.emit(e); petEvents.on(e, fn);
```

业务侧接入(每处一行):

| 位置 | 接入 |
|---|---|
| `App.tsx handleSave()` 成功后(~2090) | `emit("saved")` |
| `App.tsx handleCheckpoint()` 成功后(~2112) | `emit("checkpoint")`;push 成功回调 → `emit("pushed")` |
| `App.tsx handlePreviewRendered()`(~1940) | `emit("compiled")` |
| `LatexPreview` 错误分支 | `emit("compile-error")` |
| `handleRestore()`(~2157) | `emit("restored")`(猫表现为惊讶地抖一下) |
| `CodeEditor onChange`(~2239) | 节流 1s `emit("typing")` |

`<PetOverlay />` 挂载在 `.app` 内、modal 之后,内部动态 `import()`
three.js 场景模块;卸载时完整 dispose(geometry/texture/renderer)。

## 9. 设置与持久化

- 设置面板新增「宠物猫」分组:开关(默认**开**)、呼噜声开关(默认关)。
  关闭 = 卸载场景并释放 WebGL 上下文,非隐藏。
- `localStorage`:`rv-pet-on`、`rv-pet-corner`、`rv-pet-sound`。
- **reduced-motion**:猫退化为静态坐姿(单帧渲染,rAF 停止),保留
  眨眼都不做;庆祝退化为气泡文案(`✓ 已保存,喵`)出现 2s。i18n:
  气泡文案走 `i18n.ts`(`喵?` / `mew?` 等,中英各备)。

## 10. 性能红线

1. **渲染预算**:200×240 画布、`devicePixelRatio` 封顶 2;一盏
   hemisphere + 一盏 directional,**无 shadow map**(毛发壳层自带的
   AO 渐变已提供体积感);壳层 16 层一次 instanced draw,目标单帧
   GPU < 1.5ms。
2. **rAF 节流**:`sleep`/`idle` 降至 12fps(逐帧时间片跳帧);`watch`/
   `petted`/`celebrate` 满 60fps;`document.hidden` 或窗口失焦时暂停
   渲染(Tauri 桌面常驻,这条必须有)。
3. **输入零回归**:`typing` 事件路径上不允许任何同步三维计算(事件只
   置 dirty flag,渲染循环里消费);编辑器输入延迟与无猫时无差异。
4. 模型 + 贴图 + three.js 增量 ≤ 1.8MB(gzip);全部懒加载,不进主 chunk。
5. 内存:关闭宠物或组件卸载后 `renderer.dispose()` + 纹理/几何释放,
   重复开关 10 次无泄漏(Instruments 抽查)。
6. **WebGL 兜底**:上下文创建失败或 `webglcontextlost` 不可恢复时,
   静默隐藏宠物并在设置项旁标注"当前环境不支持",不弹错误。

## 11. 资产清单(cat.glb 要求)

- 写实比例家猫,≤ 25k 三角面,PBR 贴图 ≤ 1024²(diffuse/normal),
  另备 256² 毛发 alpha 噪声图(壳层用)。毛色:暖棕虎斑。
- 骨骼:含独立头骨、双眼骨(look-at 必需)、耳 ×2、尾链 ≥ 4 节。
- 动画 clip(命名固定):`Idle_Sit`、`Sleep`、`WakeStretch`、`Groom`、
  `PawTap`、`TailChase`、`Petted`、`Jump`、`HeadTilt`、`Carried`。
- 来源:优先 Sketchfab 可商用授权(CC-BY 需在 About 面板署名)或采购;
  **资产到位前不开工实装**——占位的低模猫会毁掉"真实风格"的目标,
  宁可没有。

## 13. 验收标准

1. **真实感**:正常观看距离下能看到猫身上的毛发层次(侧光下毛边缘
   有可辨认的绒毛轮廓),不是光滑贴图猫。
2. **视线**:移动鼠标,猫头与眼平滑跟随,眼动快于头动;头部偏转不
   超过解剖学合理范围,无骨骼穿插。
3. **互动**:在猫身上往复滑动进入抚摸态(眯眼蹭手);猫以外区域的
   全部 UI 点击不受任何遮挡;拖拽猫可换四角且重启后记住位置。
4. **键盘**:在编辑器连续打字,猫转头注视编辑器侧、尾尖随节奏摆动;
   停止 4s 回 idle;5 分钟无操作入睡,再次输入伸懒腰醒来。
5. **邀请**:连续使用 30 分钟内观察到 ≤ 3 次自发行为;modal 打开、
   编译错误存在期间零邀请;无响应后频率可感知地下降。
6. **庆祝**:手动保存 → 小庆祝;打 checkpoint → 起跳 + 画布内光尘;
   60s 内重复保存仅得到眨眼;庆祝粒子不越出猫的画布。
7. **性能**:1100×720 + 长文档编辑下,开猫前后输入延迟无可感知差异;
   idle 时 CPU/GPU 占用接近无猫基线(rAF 已降频);失焦后渲染暂停。
8. **开关与降级**:设置关闭后 WebGL 资源完全释放;reduced-motion 下
   猫为静态坐姿、庆祝变文案气泡;WebGL 不可用时静默隐藏。
9. `npm run build` 通过;three.js 及模型不在首屏 chunk 中。

## 14. 不做(out of scope)

- 猫在整个窗口里自由走动 / 趴在标题栏上——遮挡与命中复杂度爆炸,
  角落常驻已满足陪伴感;后续想要再另立 spec。
- strand-based 真毛发物理(发丝模拟)——实时端成本不值,壳层已够。
- 猫叫声 / TTS / 和猫聊天(AI 人格)——本 spec 只做无声陪伴
  (呼噜声除外,默认关)。
- 多只猫、换肤、喂食养成系统。
- 移动端 / Web 端适配(本产品是桌面 Tauri 应用)。
