# Spec: 3D 星体粒子背景 ——「墨夜星尘」(Ink Nebula)

- 日期:2026-06-10
- 前置:`spec/2026-06-10-ui-redesign-letterpress.md`(token 体系)、
  `spec/2026-06-10-motion-polish.md`(已实装)
- 目标:为单调的应用背景加入带纵深感的 3D 星体粒子 + 星云雾效果,
  让产品有"深夜星空下的工坊"氛围——同时不打架:不抢内容、不费电、
  不在工作区(编辑器)里晃眼。
- 状态:**已实装(2026-06-11)。** 调试工具:`nebula-dev.html`
  (`?theme=dark|light`,仅 dev server 可访问,不进产物)。实装备注:
  雾团精灵在低 alpha 下出现径向色带,已用高斯式渐变落点 + 烘焙抖动
  噪声(幅度 ≈ 合成后一个量化级)解决;浅色主题保留默认开。
- **设计冲突声明**:motion-polish spec §5 曾把"idle 循环动画/粒子"列为
  不做项。本 spec 按产品决策将背景星云设为**唯一的品牌例外**,以下列
  纪律换取豁免:透明度封顶、运动极慢、编辑器视图禁用、可关闭、
  reduced-motion 降级为静态。该例外仅此一处,其余界面仍"安静直到出事"。

---

## 1. 设计意图(避免"AI 粒子背景"俗套)

通用粒子库的默认效果(白点 + 连线、鼠标斥力、高速漂移)是最廉价的
"科技感"模板,与本产品无关。我们的叙事是**深夜工坊窗外的星空**:

- **星尘(粒子)**:黄铜色尘埃微粒,像台灯光柱里的浮尘 + 远处的星,
  色相全部来自品牌 token(brass 80°、ink-blue 245°),不引入新颜色。
- **星云(雾)**:两三团极低饱和的墨蓝/黄铜辉光,缓慢呼吸,像哈勃照片
  调到 5% 不透明度——感知上是"背景有了温度",而不是"背景在表演"。
- **3D 纵深**:粒子分布在一个透视投影的体积里,近大远小、近亮远暗,
  缓慢向观察者漂移(星野穿行感,周期以分钟计),配合指针的 ±6px 视差,
  纵深感来自投影而非蛮力数量。
- **克制铁律**:任何一帧,背景整体视觉权重不得超过"注意到了但不看它"
  ——所有发光元素 alpha 封顶见 §4 参数表。

## 2. 放置与生命周期

- **渲染层**:`<canvas class="nebula">` 固定定位,位于 `.app` 背景之上、
  `.content` 之下(z-index 介于 0 与 nav 之间,新 token `--z-nebula: 1`;
  `.content`/`.navbar` 需确认有自己的堆叠上下文)。`pointer-events: none`。
- **出现的视图**:home、category、assets——这些视图有大量空背景。
  **version(编辑器)视图禁用并完全停帧**:工作区需要安静,性能让位给
  编译与 PDF 渲染。
- **停帧规则**(rAF 循环暂停,非仅隐藏):
  1. 进入 version 视图;
  2. `document.visibilityState !== "visible"`(最小化/切走);
  3. 窗口失焦(`blur` 事件)→ 降帧到 12fps 而非全停(避免切回瞬间跳变),
     聚焦恢复 60fps;
  4. 设置中关闭(§6);
  5. `prefers-reduced-motion` → 渲染一张静态帧(星尘随机分布 + 星云
     定格),不进动画循环。
- modal 打开时不停帧(背景被 backdrop 压暗,继续极慢漂移无干扰)。

## 3. 技术方案(零依赖,Canvas 2D + 手写透视投影)

**否决 three.js / WebGL**:一个背景效果引 600KB 依赖不成比例;粒子量
(≤ 320)与效果(精灵贴图合成)在 Canvas 2D 上 2ms/帧以内可达。

### 3.1 粒子模型(`src/nebula.ts`,~250 行)

```ts
interface Star {
  x: number; y: number; z: number;  // 世界坐标,z ∈ [zNear, zFar]
  hue: "brass" | "ink";             // 9:1 偏黄铜
  size: number;                      // 基础半径(投影前)
  twinklePhase: number;              // 闪烁相位
}
```

- 透视投影:`screenX = cx + (x / z) * f`,`f` 为焦距常数;
  半径与 alpha 均乘 `zNear / z`(近大远小、近亮远暗)。
- 运动:每帧 `z -= speed * dt`(向观察者极慢漂移);`z < zNear` 时在
  zFar 重生(换新 x/y)——星野穿行。速度档:穿越全程 ≈ 90s。
- 闪烁:`alpha *= 0.85 + 0.15 * sin(t * 0.8 + twinklePhase)`——慢速微闪,
  不做频闪。
- 视差:指针位置经低通滤波(lerp 0.05)映射为相机偏移 ±6px,作用于
  投影中心;无指针(触控板未动)时无偏移。
- **绘制用预渲染精灵**:启动时离屏 canvas 画两种发光圆点(径向渐变,
  brass / ink 两色 × 3 档尺寸 = 6 张精灵),主循环只做 `drawImage`,
  避免每帧每粒子构建渐变(性能关键)。
- 合成模式:深色主题 `globalCompositeOperation = "lighter"`(加色发光);
  浅色主题用默认 `source-over`(加色在白底上不可见)。

### 3.2 星云雾层

- 3 团预渲染的大尺寸径向渐变精灵(≈ 视口 60% 尺寸,墨蓝 ×2、黄铜 ×1),
  各自以不同周期(45s / 70s / 100s)做极慢的位移 + 缩放呼吸
  (scale 0.95↔1.05)+ 整体 alpha 呼吸。
- 在粒子之下绘制;模糊感烘焙在精灵渐变里,**不用 ctx.filter = blur**
  (实时模糊在 2D canvas 上昂贵)。

### 3.3 画布管理

- 尺寸跟随容器(ResizeObserver),`devicePixelRatio` 封顶 2。
- 粒子数按面积密度:`min(320, area / 4800)`(1100×720 ≈ 165 颗;全屏
  4K 也不过 320)。
- React 接入:`<NebulaCanvas />` 组件(`src/NebulaCanvas.tsx`),挂载/
  卸载管理 rAF 与 ResizeObserver;App 按 view.kind 条件渲染。
- 时间步进用 `dt` 累积(rAF 时间戳),停帧恢复后无跳变。

## 4. 主题适配参数表

| 参数 | 深色(主场) | 浅色 |
|---|---|---|
| 星尘 alpha 上限 | 0.55(近端最亮粒) | 0.30 |
| 星尘颜色 | brass `oklch(0.8 0.115 83)` / ink `oklch(0.72 0.09 245)` | bronze `oklch(0.5 0.11 78)` / ink `oklch(0.48 0.1 250)`(白底上用深色细尘,"纸上墨点"而非发光) |
| 星云 alpha 上限 | **0.06** | **0.035** |
| 合成模式 | lighter | source-over |
| 粒子尺寸 | 0.5–1.8px(投影后) | 0.4–1.2px |

主题判定复用 `matchMedia("(prefers-color-scheme: dark)")`(与 CodeEditor
同模式),切换时重建精灵。浅色主题整体效果定位为"纸面细尘",若实装后
浅色观感不佳,允许降级决策:浅色默认关闭(参数表保留)。

## 5. 可读性保障

- 卡片、modal、工具条均为不透明 `--surface`,内容可读性不受影响;
  唯一直接坐在背景上的文字是空状态/`.placeholder` 文案——为其加
  `text-shadow: 0 1px 8px var(--bg)` 或半透明底衬,确保对比度仍 ≥ 4.5:1
  (以背景最亮可能帧计算)。
- 星云团中心避开视口正中(内容主区),锚定在四角偏外位置。

## 6. 设置开关

- SettingsModal 外观区新增一行:`背景星云`(`segmented`:开 / 关),
  localStorage `rv.nebula`,默认**开**。
- 关闭 → 组件卸载,画布移除;打开 → 重新挂载,粒子重新随机分布
  (无需持久化粒子状态)。
- i18n key:`nebula_bg` / `nebula_on` / `nebula_off`。

## 7. 性能预算(实装红线)

1. 主循环 CPU ≤ **2ms/帧**(M 系列基线),粒子 320 上限不可调高;
   测量:循环内 `performance.now()` 采样,DevTools Performance 复核。
2. 内存:精灵 ≤ 6 张小图 + 3 张星云图,总位图 < 8MB。
3. 不在循环内创建对象(粒子数组预分配复用),避免 GC 抖动。
4. 编辑器视图停帧后,Activity Monitor 中 App 的 GPU/CPU 占用应回到
   纯静态水平(验收项)。
5. 画布上禁用 `ctx.filter`、阴影(shadowBlur)——全部预烘焙。

## 8. 验收标准

1. **氛围**:home 视图深色主题下,可见缓慢漂移的星尘与隐约星云,纵深
   感明确(近粒明显大于远粒、移动快于远粒);截图静帧给陌生人看,
   第一反应是"背景有质感",而非"屏幕上有粒子动画"。
2. **纪律**:任意时刻暂停截帧,星云区域取色 alpha 合成后与纯 `--bg` 的
   对比 ≤ 参数表上限;卡片/文字可读性无感知差异。
3. **生命周期**:进入编辑器视图 → rAF 完全停止(日志/Performance 验证);
   返回 home → 恢复且无跳变;最小化停帧;失焦降帧。
4. **视差**:指针移动时背景反向轻移(≤ 6px),平滑无抖动;指针静止
   背景仍自漂移。
5. **reduced-motion**:系统开启后为静态星空(单帧),无任何循环。
6. **开关**:设置关闭后画布消失,重启保留偏好;开启即恢复。
7. **性能**:满网格 home 视图滚动 + 星云运行,无掉帧;循环 CPU ≤ 2ms/帧;
   编辑器视图占用回落。
8. `npm run build` 通过;浅色主题观感评审(保留降级为"浅色默认关"的
   决策权)。

## 9. 不做(out of scope)

- three.js / WebGL shader 星云(体积雾、辉光后处理)——效果上限更高
  但依赖与功耗不成比例;若未来要做"启动画面"级演出再评估
- 鼠标交互粒子(斥力、连线、点击爆裂)——廉价科技感模板,明确拒绝
- 粒子出现在编辑器 / PDF 预览区
- 星座连线、流星等具象元素
- 按电池状态自动降级(`navigator.getBattery` 已废弃,停帧规则已覆盖
  主要场景)
